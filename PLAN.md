## HOW TO WORK IN THIS REPO

**Edit source files only — never edit `insposearch/app.js` directly.**

```
src/app.js       ← UI logic, search orchestration, event handlers
src/state.js     ← all constants, source registry, query classifiers
src/fetchers.js  ← one fetch function per source API
src/core.js      ← shared utilities (health, scoring, cache, etc.)
src/i18n.js      ← translation runtime
insposearch/     ← built output (auto-generated by build step)
```

After any edit to `src/`:
```bash
npm run build    # runs update-source-count.cjs + esbuild → insposearch/app.js
npm start        # serve locally at http://localhost:3000
```

Run tests: `npm test`

## TASK GROUP 2 — Source Scoring & Relevance
> **Priority: High.**  
> File to edit: `src/core.js` (scoring), `src/state.js` (constants)

### 2a. Add `artist` field to relevance scoring

Currently `scoreItemRelevance()` in `src/core.js:611` scores against `title`, `description`, and `tags`. The `artist` field is NOT included, so artist-name searches score poorly.

**What to do:**  
In `src/core.js` around line 611, find `scoreItemRelevance`. Inside the haystack string construction, add `item.artist` alongside the existing fields. Example — look for a line like:
```js
const hay = `${item.title} ${item.description} ${(item.tags||[]).join(' ')}`.toLowerCase();
```
Change to:
```js
const hay = `${item.title} ${item.description} ${item.artist||''} ${(item.tags||[]).join(' ')}`.toLowerCase();
```
Do this everywhere `hay` is constructed for scoring in that function.

### 2b. Boost exact title-match score

When the query appears verbatim in the image title, the score should be higher than when it only appears in description. Currently all field matches score equally.

**What to do:**  
In `scoreItemRelevance` in `src/core.js`, after computing the haystack score, add a bonus multiplier when the query appears in `item.title` specifically:
```js
const titleHay = (item.title || '').toLowerCase();
if (titleHay.includes(queryLower)) score += 5; // title bonus
```
Check what the existing scoring structure looks like before adding — find the `score` variable and add the bonus near the end before `return score`.

### 2c. Penalize items with no title

Items where `item.title` is null, empty, or a generic filename (e.g. `IMG_4523`, `Untitled`) add noise to results. They should be penalized in scoring, not just filtered in exact mode.

**What to do:**  
In `scoreItemRelevance`, near the start, add:
```js
if (!item.title || item.title.trim().length < 3) score -= 3;
```
